# OJ平台接口文档

### Token 版本号机制

为保证同一账号仅单设备在线，系统采用 Token 版本号机制：

- `Users` 表中维护 `token_version` 字段（默认 `0`），每次登录或修改密码时自动 `+1`。
- JWT Token 的 Payload 中包含 `token_version`，每次请求时服务端会校验 Token 中的版本号与数据库是否一致。
- 版本号不匹配时，说明账号已在其他设备登录或密码已变更，服务端返回 `401`，前端需跳转至登录页。

## 1 用户相关接口

###  1.1 注册

| 项目         | 内容                 |
| :----------- | :------------------- |
| **请求路径** | `/user/register`     |
| **请求方式** | `POST`               |
| **接口描述** | 该接口用于注册新用户 |

#### 请求参数

请求参数格式：application/json

| 参数名称   | 说明   | 类型     | 是否必须 | 备注                       |
| ---------- | ------ | -------- | -------- | -------------------------- |
| `username` | 用户名 | `string` | 是       | 4~20位字母/数字/下划线     |
| `password` | 密码   | `string` | 是       | 6~32位，建议包含字母和数字 |
| `email`    | 邮箱   | `string` | 否       | 有效的邮箱格式             |

**请求数据样例：**

```shell
{
    "username": "little_coder",
    "password": "SecurePass123",
    "email": "coder@example.com"
}
```

#### 响应数据

响应参数格式：application/json

响应参数说明：

| 名称    | 类型   | 是否必须 | 默认值 | 备注                  | 其他信息 |
| ------- | ------ | -------- | ------ | --------------------- | -------- |
| code    | number | 必须     |        | 响应码, 0-成功,1-失败 |          |
| message | string | 非必须   |        | 提示信息              |          |
| data    | object | 非必须   |        | 返回的数据            |          |

**响应数据样例：**

```json
// 成功
{
    "code": 0,
    "message": "注册成功",
    "data": {
        "user_id": 1001,
        "username": "little_coder"
    }
}
// 失败
{
    "code": 1,
    "message": "用户名已存在",
    "data": null
}
```



### 1.2 登录

| 项目         | 内容                                                         |
| :----------- | :----------------------------------------------------------- |
| **请求路径** | `/user/login`                                                |
| **请求方式** | `POST`                                                       |
| **接口描述** | 该接口用于用户登录，返回JWT Token。登录成功后，同一账号其他设备上的 Token 将失效。 |

#### 请求参数

请求参数格式：`application/json`

| 参数名称   | 说明   | 类型     | 是否必须 | 备注             |
| :--------- | :----- | :------- | :------- | :--------------- |
| `username` | 用户名 | `string` | 是       | 注册时的用户名   |
| `password` | 密码   | `string` | 是       | 注册时设置的密码 |

**请求数据样例：**

```json
{
    "username": "little_coder",
    "password": "SecurePass123"
}
```

#### 响应数据

响应参数格式：`application/json`

| 名称              | 类型     | 是否必须 | 默认值 | 备注                       | 其他信息 |
| :---------------- | :------- | :------- | :----- | :------------------------- | :------- |
| `code`            | `number` | 必须     | -      | 响应码，`0`=成功，`1`=失败 | -        |
| `message`         | `string` | 非必须   | -      | 提示信息                   | -        |
| `data`            | `object` | 非必须   | -      | 包含token和用户信息        | -        |
| `data.token`      | `string` | 必须     | -      | JWT访问令牌                | -        |
| `data.expires_in` | `number` | 必须     | -      | 令牌有效期（秒）           | -        |
| `data.user`       | `object` | 必须     | -      | 用户基本信息               | -        |

**响应数据样例：**

```json
// 成功
{
    "code": 0,
    "message": "登录成功",
    "data": {
        "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
        "expires_in": 86400,
        "user": {
            "user_id": 1001,
            "username": "little_coder",
            "avatar": "https://example.com/avatars/1001.jpg",
            "role": "user",
            "solved_problems": 42,
            "total_submissions": 128
        }
    }
}
// 失败
{
    "code": 1,
    "message": "用户名或密码错误",
    "data": null
}
```

### 1.6 更新用户信息

| 项目         | 内容                                                         |
| :----------- | :----------------------------------------------------------- |
| **请求路径** | `/user/{user_id}`                                            |
| **请求方式** | `PUT`                                                        |
| **接口描述** | 更新当前登录用户的用户名或邮箱。修改用户名后 Token 仍然有效，无需重新登录。 |

#### 请求参数

请求参数格式：`application/json`

> **注意**：至少提供一个字段。如果两个字段都不传，返回参数错误。

**请求数据样例：**

```json
{
    "username": "new_coder_name",
    "email": "new_email@example.com"
```

**响应数据样例：**

```json
// 成功
{
    "code": 0,
    "message": "更新成功",
    "data": {
        "user_id": 1001,
        "username": "new_coder_name",
        "email": "new_email@example.com",

```

### 1.7 上传/修改用户头像

| 项目         | 内容                                                         |
| :----------- | :----------------------------------------------------------- |
| **请求路径** | `/user/{user_id}/avatar`                                     |
| **请求方式** | `PUT`                                                        |
| **接口描述** | 上传或修改当前登录用户的头像。头像文件将上传至服务器，返回可访问的 URL。仅本人可操作。 |

#### 请求参数

请求参数格式：`multipart/form-data`

| 参数名称 | 说明         | 类型   | 是否必须 | 备注                                                         |
| :------- | :----------- | :----- | :------- | :----------------------------------------------------------- |
| `avatar` | 头像图片文件 | `file` | 是       | 支持 `jpg` / `png` / `gif` / `webp`，建议大小不超过 5MB，建议尺寸 200x200 至 1024x1024 |

**请求数据样例（表单）：**

```text
PUT /user/1001/avatar
Content-Type: multipart/form-data

avatar: @avatar.jpg
```

#### 响应数据

响应参数格式：`application/json`

| 名称      | 类型     | 是否必须 | 默认值 | 备注                       | 其他信息 |
| :-------- | :------- | :------- | :----- | :------------------------- | :------- |
| `code`    | `number` | 必须     | -      | 响应码，`0`=成功，`1`=失败 | -        |
| `message` | `string` | 非必须   | -      | 提示信息                   | -        |
| `data`    | `object` | 非必须   | -      | 返回新的头像 URL           | -        |

**响应数据样例（成功）：**

```json
// 成功
{
    "code": 0,
    "message": "头像更新成功",
    "data": {
        "user_id": 1001,
        "avatar": "https://example.com/avatars/1001.jpg"
    }
}
// 失败 - 文件格式不支持
{
    "code": 1,
    "message": "不支持的文件格式，仅支持 jpg/png/gif/webp",
    "data": null
}
// 失败 - 文件过大
{
    "code": 1,
    "message": "文件过大，请上传不超过 5MB 的图片",
    "data": null
}
```

### 4.2 修改题目
